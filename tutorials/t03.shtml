Pointers

The C files in the starter code contain main() functions that process input from the command line and call a helper function. Your job, in both cases, is to complete the helper function so that it works without changing the main() function.

  1. invest.c: Implement a function invest() that takes an amount of money and multiplies it by a given rate. It's your job to figure out the exact type of this function's arguments, given the sample usage in the main() function in the starter code.
  2. score_card.c: Implement a function sum_card() that takes an array of pointers to integers and returns the sum of the integers referenced by the array. You should also free any dynamically allocated memory to avoid any errors due to memory leaks.

Using scanf()

This part does not have starter code. Your job is to write a program that reads phone numbers from standard input using scanf(). This task is related to the assignment, so I encourage you to work on it in the lab, with help from your TA and peers. (Remember: The assignments are individual work. This is you chance to work with your friends before demonstrating what you can do.)

If you did not watch the PCRS video Input, Output, and Compiling: Reading Input (scanf), you may want to start by watching it.

Your program should be implemented in a file named phone.c. The program takes no command line arguments. Instead, it will read one line at a time from standard input until the stream is closed. Each line will contain two elements: a 10 character string and an integer.

If the integer is zero, the program prints the full string to standard output followed by a single newline \n character. If the integer is between 1 and 9 inclusive, the program only prints the corresponding digit from the string to standard output (again followed by a newline). After printing the requested output, the program should read and process the next line from standard input.

Once all of the input has been processed, the program should terminate and return 0.

Command-line arguments:: If the user passes in [phone number] 0, the program should print the entire phone number starting from index 0 of the phone number array. If the user passes in [phone number] N, where N is an integer between 1 and 9, the program should print only the element at index N of the phone number array.

Reaching end-of-input:
In a UNIX terminal, you can signal the end-of-input by pressing ctrl+d on a blank line. This will signal to scanf() that there is no more input to process. Also see man scanf and search for EOF.

Validating Input Length

Writing your program under the assumption that the input will always follow a certain set of rules (e.g., assuming that a user will never type in a phone number that is more than 10 digits in length, or that they will only type in numbers, but no other characters) is a major cause of bugs, and security vulnerabilities in particular. Improper input validation ranks third place in the CWE Top 25 Most Dangerous Software Errors. Let us focus here on validating the length of the input.

scanf() allows you to specify a field width in the conversion specifier. For example, the conversion spcifier %16c (if you use %c, make sure you understand the difference between that and %s, explained in man 3 scanf) tells scanf() to read a series of up to 16 characters into your character array (i.e., any additional characters are ignored). This helps you to avoid buffer overflow, in case your character array is not large enough to store all the input characters (buffer overflow ranks first place in the above-linked top 25 most dangerous software errors).

With this knowledge, update your program phone.c to place a limit on the maximum number of characters that scanf() should read. In the event that oversized input is provided by the user (whether for the phone number string or the integer that follows it), your program should not make any unsafe memory accesses, i.e., it should run without crashing when compiled with the -fsanitize=address flag. Ideally your program should print any one-line message (or an error message) followed by a single newline \n character, and proceed to process the next line from standard input. However, you might find instead that your program begins to give incorrect results for subsequent values that you enter—you don't need to worry about that for this tutorial (as long as your program doesn't segfautl), but if you are curious about why that happens you may check out this page and this post on the forum.

Assignment Tip: More On Input Validation

You might have noticed that scanf() format specifiers can be difficult to deal with. For Assignment 1, it might be easier to simply read a whole line of input, either with scanf("%Ns", str) (where N+1 is the size of your array) or fgets(), and to use strtol() to read the individual numbers that you need. But either approach can be used, if done correctly (i.e., no unsafe memory accesses should be made, and there is no "correctness" requirement in how to process invalid input, other than terminating gracefully).
