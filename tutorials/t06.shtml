Tutorial 6: Make

Introduction

This lab is designed to give you some practice using make and incorporating bash scripting into your Makefiles. You may wish to look at the Makefiles for Assignments 1 and 2 for inspiration and examples.

Your Task

The t06 directory in your repository contains a small collection of source, header, and test files. Nothing particularly interesting is in the files. They are dummy programs written to give you a project to compile. Much like Assignment 2, two of the source files contain main functions, so your task is to write a Makefile that compiles the two executables and verifies that they perform as expected on the test file.

You may modify the given starter files to help you test, but we are not expecting you to submit any C code in this lab. The Makefile is the only deliverable.

The two executables are to be named serialize (built using serialize.c and helpers.c) and deserialize (built using deserialize.c and helpers.c). Several of the .c files rely on helpers.h. Both executables read from stdin and do not expect any arguments. Both emit output to stdout. If they complete successfully, they will exit with status code 0; if there is an error, they will exit with a non-zero status code.

Basic test input for the serialize program is in test.dat. deserialize is expecting input in the same format as serialize's output, so you can save the output of serialize to use as input for deserialize.

Requirements

Your Makefile has several required targets: serialize and deserialize, which should create the specified executables using separate compilation; basic_test, described below; and clean, which should remove all of the created executables and intermediate files. If make is invoked without an argument, basic_test should be executed.

basic_test should update the executables, if necessary, and then run serialize on the input in test.dat and deserialize on the output of serialize to verify that they both complete successfully. If both programs execute successfully (regardless of the output generated), then "Basic test passed" should be printed. If one or both programs fail to execute, then "Basic test failed" should be displayed.

Tips

  1. Each action in a rule must be preceded by a tab—not spaces.
  2. Make sure to use separate compilation. Compile the .c files into .o files and then link the .o files.
  3. Include header files as dependencies for the files that include them.
  4. @ can be used to keep Make from echoing the commands it is executing.
  5. Your action may include "automatic" make variables, which are expanded by the make program before executing the action. See the GNU make manual for a full list of automatic variables.
  6. If your action includes bash variables, the $ will need to be escaped ($$) so that it is not expanded by make (so that it is instead expanded by Bash).
  7. In the Bash shell, the variable $? contains the status code for the last process that was executed. See Section 3.2.5: Special Parameters in Bash Guide for Beginners

What to Submit

We'll be looking for a Makefile in your t06 directory. Please make sure it has been added, committed, and pushed. We will verify that the Makefile has the required rules and compiles both executables correctly (1 point), rebuilds the minimum number of targets necessary (1 point), and implements the basic test correctly (1 point).
