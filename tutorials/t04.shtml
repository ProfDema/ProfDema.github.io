Valgrind

Valgrind is a set of tools for debugging and profiling code. It can be used to find errors, to detect vulnerabilities, and to optimize performance. I use it frequently as part of my development process. Whenever I develop significant C code, I run several of the Valgrind tools before and after testing.

For now, we'll focus on a single tool from the suite: memcheck. You first task is to familiarize yourself with the tool using the quick start guide. Add the error report generated by Valgrind when run on the example in the tutorial to your repository in the text file valgrind_example.txt.

Now for a challenge. Take a look at ll_sum.c, an ugly bit of code that places input from the user into a linked list (you may refer back to Linked Structures and Iteration in PCRS Week 4 to review linked lists) and then iterates over the list to compute a sum. Your objective is to fix the code so that Valgrind emits no errors (about uninitialized values, in this case) and detects no leaked heap memory. You should not rewrite the code to use an array or to sum the values as they come; the original author might be crazy, but they want to try out linked lists. Commit your repaired code to the repository.

Finally, try running Valgrind on phone.c, which is related to a problem from last week. There is a small error in this code. Does Valgrind find it? Can you? This illustrates that Valgrind is not perfect. It can give you hints as to what is wrong, but it won't be able to find every error.

Strings Practice

We have provided C files for four string problems: compare.c, truncate.c, strip.c, and greeting.c. Implement each of these four programs according to the comment inside the starter file. Most will only take a few lines of code and rely on functions in strings.h. I recommend that you solve these problems in the order in which they were listed.

Address Sanitizer vs. Valgrind

Note that the compiler's Address Sanitizer (-fsanitize=address) will catch memory leaks, just like Valgrind. However, it will not catch read operations on uninitialized variables. So, running your code through Valgrind is crucial for ensuring that your program doesn't read and rely on any garbage (uninitialized) values, even if the Address Sanitizer doesn't result in errors. If you're interested in reading a blog post about some real-world advanced usage of Valgrind, check out this blog post: Adventures making gnome-shell run in Valgrind.
